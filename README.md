# Java_-coursework_db
## Разработка клиент-серверного приложения «Система учёта производственных операций и нормирования сборки», предназначенного для хранения, обработки и анализа информации о составе изделий, операциях их сборки и связанных с этими операциями нормах трудовых затрат.

Проект включает в себя как серверную, так и клиентскую части, что позволило продемонстрировать комплексное применение современных технологий разработки в контексте производственного учёта.

На стороне сервера, с использованием PostgreSQL и языка PL/pgSQL, была спроектирована и реализована реляционная база данных, соответствующая схеме связей «Нормы затрат труда <–>> Сборка <<–> Детали». Были созданы три основные таблицы (`details`, `assembly`, `labor_norms`) со всеми необходимыми ограничениями целостности (первичные и внешние ключи, `CHECK`, `GENERATED ALWAYS AS IDENTITY`, `NOT NULL`). Для обеспечения сложной бизнес-логики, безопасности данных и повышения производительности были разработаны дополнительные объекты базы данных: представление (`v_detail_assembly_full`), хранимая процедура с выходными параметрами (`get_detail_production_stats`), скалярная (`get_operation_total_time`) и табличная (`get_assembly_details`) функции. Особое внимание было уделено реализации различных стратегий поддержания ссылочной целостности: одна связь реализована с использованием системного каскадного обновления (`ON UPDATE CASCADE`), а для другой связи создан специализированный триггер `labor_norms_cascade_delete`, обеспечивающий каскадное удаление и дополнительный триггер `check_worker_grade`, контролирующий соответствие квалификации рабочего требованиям операции.

На стороне клиента было разработано полнофункциональное кроссплатформенное десктоп-приложение на языке Java в среде Vscode. Приложение построено на принципах архитектуры, близкой к MVC, с чётким разделением ответственности между моделью (POJO-классы `Detail`, `Assembly`, `LaborNorm`), слоем доступа к данным (DAO-классы) и представлением (Swing-панели). Для взаимодействия с СУБД PostgreSQL использовалась технология JDBC.

Практическая часть работы включала решение трёх аналитических задач, каждая из которых демонстрирует различные методологические подходы к обработке данных:

1. Анализ состава изделий был реализован двумя способами: с использованием серверной табличной функции `get_assembly_details` (эффективная выборка на стороне СУБД) и посредством загрузки данных в коллекции Java-объектов с последующей фильтрацией средствами Stream API (наглядная клиентская обработка). Это позволило сравнить производительность и семантику двух подходов.

2. Анализ норм времени с оконными функциями был выполнен с помощью мощных средств PostgreSQL — агрегатных функций с предложением `OVER (PARTITION BY)`. Это позволило рассчитать доли подготовительного и штучного времени, а также определить максимальное штучное время для каждой детали в рамках единого запроса, значительно упрощая аналитическую логику клиентского приложения.

3. Решение задачи с квантором общности («для всех») было представлено в двух парадигмах:
   - Декларативное SQL-решение через конструкцию с двойным `NOT EXISTS`, что является каноническим способом выражения кванторной логики в реляционных СУБД.
   - Императивное Java-решение через record-ориентированный обход коллекций с помощью вложенных циклов, что наглядно демонстрирует процедурную реализацию логики «для каждой детали… для каждой операции… проверить существование…».

Клиентский интерфейс предоставляет пользователю удобные инструменты для работы: главную форму с навигацией Master-Detail между связанными таблицами, альтернативную форму с динамическим переключением между источниками данных, а также специализированные панели для выполнения аналитических запросов с возможностью ввода параметров и выбора метода решения.

Таким образом, выполненный проект не только решает конкретную предметную задачу автоматизации учёта производственных операций, но и служит комплексным учебным примером, охватывающим полный цикл разработки информационной системы: от проектирования реляционной базы данных на PostgreSQL и реализации бизнес-логики на стороне сервера до создания полнофункционального кроссплатформенного клиентского приложения на Java с графическим интерфейсом. Работа демонстрирует владение как декларативными (SQL), так и императивными (Java) средствами программирования, а также понимание принципов построения многоуровневых клиент-серверных архитектур.
